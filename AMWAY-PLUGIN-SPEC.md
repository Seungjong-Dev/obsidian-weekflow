# AmwayFlow 플러그인 상세 기획안: FU/FT 중심 설계

## 핵심 철학

> "Fortune is in the Follow-Up" — 암웨이 사업의 80%는 팔로업이다.

대부분의 ABO가 실패하는 이유는 제품이나 플랜이 부족해서가 아니라, **체계적인 팔로업 시스템이 없어서**다. 이 플러그인은 프로스펙팅부터 사업자 육성까지의 **전체 FU/FT 파이프라인**을 자동화한다.

---

## 1. FU/FT 파이프라인 정의

### 1.1 전체 파이프라인 (7단계)

```
[1. 발굴]  →  [2. 첫 컨택]  →  [3. 약속]  →  [4. 플랜 제시]
    ↓              ↓              ↓              ↓
  Name List    Ice Breaking    미팅 확정     사업/제품 설명
                                                ↓
[7. 팀 빌딩]  ←  [6. 등록/구매]  ←  [5. 후속 FU]
    ↓                 ↓               ↓
  교육/미팅       ABO등록 or        질문 답변
  동행/멘토링     제품 구매          결정 유도
```

### 1.2 각 단계별 FU 액션 정의

| 단계 | 트리거 | FU 액션 | 기본 FU 주기 | 최대 FU 횟수 |
|------|--------|---------|-------------|-------------|
| 1. 발굴 | 이름 추가 | 관계 형성 시작 | 즉시 | - |
| 2. 첫 컨택 | 첫 대화 완료 | 관심사 기반 재컨택 | 2-3일 | 3회 |
| 3. 약속 | 약속 확정 | 약속 리마인드 | D-1, 당일 오전 | 2회 |
| 4. 플랜 제시 | 플랜 미팅 완료 | 48시간 룰 FU | 24시간, 48시간 | 2회 |
| 5. 후속 FU | 관심 표현 | 결정 유도, 의문 해결 | 2-3일 | 5회 |
| 6. 등록/구매 | 등록 or 구매 완료 | 첫 경험 지원 | 1일, 3일, 7일 | 3회 |
| 7. 팀 빌딩 | ABO 활동 시작 | 동행, 교육, 미팅 참석 유도 | 매주 | 지속 |

## 2. 프로스펙트 노트 상세 설계

### 2.1 마크다운 형식

```markdown
---
type: amway/prospect
name: 이서연
phone: "010-9876-5432"
relationship: 대학동기
interest: health           # health | beauty | income | freedom | product
temperature: warm          # hot | warm | cold | lost
stage: 플랜제시             # 발굴 | 첫컨택 | 약속 | 플랜제시 | 후속FU | 등록 | 팀빌딩
next_fu: 2026-03-10
fu_count: 3
referred_by: "[[김민수]]"
tags: [amway/prospect]
---

## FU Log
- [x] 2026-03-08 14:00 플랜 미팅 (카페 강남점) #플랜제시
  - 반응: 긍정적, 수입 구조에 관심
  - 걱정: "주변 시선이 걱정", "시간이 될까"
  - 다음 액션: 48시간 내 FU 전화, 성공 사례 공유
  - 온도 변화: cold → warm
- [x] 2026-03-05 카카오톡 메시지 #첫컨택
  - 건강 고민 이야기 → 뉴트리라이트 소개
  - 반응: "한번 들어볼게"
  - 다음 액션: 3일 후 약속 잡기
- [x] 2026-03-01 인스타 DM #발굴
  - 건강 관련 포스팅에 댓글 → DM 대화

## Objection Handling
- "주변 시선이 걱정" → 사업이 아닌 제품 먼저 경험 제안
- "시간이 될까" → 파트타임으로 시작, 주 5시간이면 충분

## Personal Notes
- 직장: IT기업 마케팅팀 과장
- 가족: 남편 + 아들 1 (5세)
- 취미: 요가, 건강식
- 남편도 건강에 관심 → 커플 동반 미팅 가능성
```

### 2.2 FU Log 파싱 규칙

```
- [x] YYYY-MM-DD HH:MM 내용 #단계태그
  - 반응: (텍스트)
  - 걱정: (텍스트)
  - 다음 액션: (텍스트)
  - 온도 변화: (이전) → (이후)
```

파서가 추출하는 정보:
- 날짜/시간 → FU 이력 타임라인
- 단계 태그 → 파이프라인 단계 자동 업데이트
- 반응 → 감정 분석 & 요약
- 다음 액션 → 자동 FU 태스크 생성
- 온도 변화 → 프로스펙트 온도 자동 업데이트

## 3. FU 자동화 엔진

### 3.1 자동 FU 태스크 생성

프로스펙트 단계가 변경되면 자동으로 다음 FU 태스크를 생성:

```typescript
interface FURule {
  fromStage: Stage;           // 현재 단계
  triggerAction: string;      // "플랜 완료", "첫 컨택 완료" 등
  fuTasks: FUTask[];          // 생성할 FU 태스크들
}

interface FUTask {
  delayDays: number;          // 며칠 후 FU
  delayHours?: number;        // 시간 단위 세밀 조절
  action: string;             // "48시간 FU 전화"
  template?: string;          // FU 메시지 템플릿
  autoCreateInTimeline: boolean; // WeekFlow에 자동 블록 생성
}

// 예시: 플랜 제시 후 자동 FU 룰
const postPlanRule: FURule = {
  fromStage: "플랜제시",
  triggerAction: "플랜 미팅 완료",
  fuTasks: [
    {
      delayDays: 0,
      delayHours: 24,
      action: "플랜 FU 전화 - 궁금한 점 확인",
      template: "어제 미팅 감사합니다. 혹시 궁금하신 점 있으셨나요?",
      autoCreateInTimeline: true,
    },
    {
      delayDays: 2,
      delayHours: 0,
      action: "48시간 FU - 결정 유도",
      template: "고민해보셨나요? 성공 사례 하나 공유드릴게요.",
      autoCreateInTimeline: true,
    },
    {
      delayDays: 7,
      delayHours: 0,
      action: "1주 FU - 제품 체험 제안",
      template: "직접 제품을 체험해보시면 확신이 생기실 거예요.",
      autoCreateInTimeline: false,
    },
  ],
};
```

### 3.2 기본 FU 룰셋 (암웨이 베스트 프랙티스)

```
[첫 컨택 완료]
  → +2일: "관심사 관련 정보 공유" (카톡/문자)
  → +5일: "만남 제안" (전화)

[약속 확정]
  → D-1: "내일 미팅 리마인드" (카톡)
  → 당일 AM: "오늘 뵙겠습니다" (카톡)

[플랜 제시 완료]
  → +1일: "24시간 FU - 궁금한 점 확인" (전화) ⭐ 가장 중요
  → +2일: "48시간 FU - 결정 유도" (전화)
  → +7일: "제품 체험 제안" (만남)
  → +14일: "근황 체크" (카톡)

[제품 구매 완료]
  → +1일: "제품 받으셨나요? 사용법 안내" (카톡)
  → +7일: "사용 후기 확인" (전화)
  → +25일: "재구매 안내" (카톡)

[ABO 등록 완료]
  → +0일: "환영! 첫 단계 안내" (만남)
  → +1일: "앱 설치 & 첫 주문 가이드" (카톡)
  → +3일: "이번 주 미팅 참석 안내" (전화)
  → +7일: "첫 주 리뷰 & 네임리스트 작성" (만남)
  → +14일: "2주차 - 첫 플랜 동행" (만남)
  → +30일: "한 달 리뷰 & 목표 설정" (만남)
```

### 3.3 FU 태스크의 WeekFlow 연동

FU 태스크 생성 시 선택적으로 WeekFlow 타임라인에 블록 자동 추가:

```markdown
## Timeline
- [ ] 14:00-14:30 FU: 이서연 48시간 FU 전화 #amway-followup [[이서연]]
```

- `#amway-followup` 태그로 WeekFlow 카테고리 자동 매핑
- `[[이서연]]` 링크로 프로스펙트 노트 바로 접근
- 블록 클릭 시 FU 기록 모달 오픈

## 10. 기술 구현 핵심

### 데이터 흐름
```
[프로스펙트 노트 .md] → [prospect-parser.ts] → [ProspectData]
                                                      ↓
[FU 룰 엔진] ← [fu-engine.ts] ← [오늘 날짜 비교]
      ↓
[오늘의 FU 태스크 목록]
      ↓                    ↓
[FU Dashboard]    [WeekFlow Planning Panel]
      ↓
[FU 모달] → [프로스펙트 노트 업데이트] → [다음 FU 자동 생성]
```

### Obsidian API 활용
- `vault.getMarkdownFiles()` → 프로스펙트/ABO 노트 스캔
- `metadataCache.getFileCache()` → frontmatter 빠른 읽기
- `vault.on('modify')` → 노트 변경 시 실시간 반영
- `workspace.registerView()` → 대시보드/파이프라인 뷰 등록

### frontmatter 기반 쿼리
```typescript
// 오늘의 FU 대상 찾기
function getTodayFUProspects(vault: Vault, app: App): ProspectData[] {
  const today = moment().format("YYYY-MM-DD");
  return vault.getMarkdownFiles()
    .filter(f => f.path.startsWith(settings.prospectFolderPath))
    .map(f => {
      const cache = app.metadataCache.getFileCache(f);
      return cache?.frontmatter;
    })
    .filter(fm => fm?.type === "amway/prospect" && fm?.next_fu <= today);
}
```
